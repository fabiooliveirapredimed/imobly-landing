# Security Policy

## Reporting Security Issues

If you discover a security vulnerability in Imobly, please do not disclose it publicly.

Report security issues privately to the Imobly team at suporte@imobly.pt.

## Protected Areas

The following parts of the platform are considered sensitive and proprietary:

- Source code
- AI systems (property posts generation, role-play training, market studies, document extraction)
- CRM architecture
- Property and client data
- Authentication systems
- Payment and subscription systems
- Edge Functions and backend logic

## Prohibited Actions

The following actions are strictly prohibited:

- Unauthorized access
- Reverse engineering
- Data extraction
- Scraping
- Source code copying
- Attempting to bypass authentication
- Attempting to access private user data
- Exploiting bugs or vulnerabilities

## Responsible Disclosure

Security researchers or users must report issues responsibly and allow reasonable time for resolution before any public disclosure.

## Legal Notice

Unauthorized access, data extraction, reverse engineering, or misuse of the Imobly platform may result in legal action.
